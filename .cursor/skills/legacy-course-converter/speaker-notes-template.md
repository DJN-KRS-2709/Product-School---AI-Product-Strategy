# Speaker Notes Template

Speaker notes are markdown files — one per module. They're a briefing memo for the instructor, not a transcript to read aloud.

## Template

```markdown
# Module N — Title
## Speaker Briefing for {Instructor Name}

### What this module is about

[2-3 paragraphs. Cover:]
- The emotional arc of this session
- What students leave with (name specific artifacts)
- How it connects to the previous module
- The traditional assumption this module breaks (name it explicitly)

### The flow and why it's structured this way

[1-2 paragraphs explaining:]
- Why slides are ordered the way they are
- Where the energy shifts happen (lecture → build → break → governance → peer audit, etc.)
- Any deliberate pedagogical choices (e.g., "We moved the build before the break so it follows its lecture immediately")

---

### Slide 1 — Title

[1-3 paragraphs. Cover:]
- The shift from the previous module to this one (e.g., "product → org")
- Session waypoints and what the output promise is
- What to tease early to keep engagement through theory sections

### Slide 2 — Agenda

[Brief. Just orientation. Name the moment people should look forward to.]

### Slide 3 — Recall from Previous Module

[What they brought. The bridge question or poll to run.
Example: "Ask the room: 'Does your company have a published AI use policy?'
Point: yes / no / sort of. Most will be 'sort of.' That shared guilt sets up the provocation."]

### Slide 4 — Provocation

[Explain each True/False claim. Name the company proof for each.
These are conversation starters, not gotchas.]

### Slide N — {Slide Title}

[Continue for every slide. Each entry covers:]
- Why this slide exists at this position in the deck
- What should happen in the room (lecture, pair work, poll, reflection)
- Connection to what came before and what follows
- For tools: "The interactive tool link is on this slide — students can open the [Tool Name]..."
- For exercises: what good vs. bad answers look like, so the instructor can coach
- For repositioned slides: "We moved this here based on [Name]'s feedback because..."

### Slide N — Bridge to Next Module

[Frame the transition. Example: "M5 closes the building phase.
M6 is about selling it — the board simulation."]
```

## Tone Rules

| Do | Don't |
|----|-------|
| "Here's why this slide exists and what you should accomplish with it" | "This slide shows three bullet points about governance" |
| Conversational paragraphs | Bullet-point lists |
| Explain thinking and intent | Narrate what's on the slide |
| "Walking Carlos through the plan over coffee" energy | Formal script or teleprompter text |
| Name real companies as examples | Use generic placeholders |
| Reference source material ("Madhavan Ramanujam's work") | Leave sources unnamed |

## Content Checklist

- [ ] Opens with `# Module N — Title` and `## Speaker Briefing for {Name}`
- [ ] Has `### What this module is about` preamble
- [ ] Has `### The flow and why it's structured this way` preamble
- [ ] Horizontal rule (`---`) before slide-by-slide notes
- [ ] Every slide has an `### Slide N — Title` entry
- [ ] Activity slides explain pedagogical reasoning
- [ ] Exercises describe what good/bad answers look like
- [ ] Interactive tools mentioned naturally when relevant
- [ ] Acronyms spelled out on first use
- [ ] Bridge to next module closes the doc
